11.00 น.-12.00 น. ผู้เข้าแข่งขันรายงานตัวที่บ้านวิทยาศาสตร์สิรินธร สวทช. จ.ปทุมธานี

13.00 น.-13.30 น. ผู้เข้าแข่งขันพร้อมกันที่ห้องประชุม : โค้ชพี่ต้น พี่โบ้ พี่อ้อ พี่แมว พี่เปี๊ยก
อธิบายกติกาการแข่งขัน : ดร.กฤษ
- From AI Model to Service on AI for Thai 
- ทุกทีมมี AI Model (เน้นวิธีการเอาไปทำเป็น Service, ไม่จำเป็นต้องพัฒนาเอง)
ผลงานที่จะต้องส่งมอบ
- การพัฒนา AI Model เป็น Service API 
- การสร้าง Application ที่ใช้งานได้
เกณฑ์การตัดสิน
- Technical Score 65% ตัดสิน 18:00 วันอาทิตย์
	- Design Report (README.md) 5% @ Checkpoint # 1
	- Postman Collection (or compatible API Testing tool) 5% @ Checkpoint # 2
	- Implement Queue 5%
	- CI/CD 5%
	- Data Management Service 5%
	- Performance & Robust 10%
		- Error Handling
		- Validation
	- Technical Challenges 25%
		- Design Complexity
	- Resource Management 5%
		- Model Size < 16GB = 5%
		- Model Size < 24GB = 2%
		- Model Size > 24GB = 0%
- Application Score 35% ตัดสินวันนำเสนอ
	- Creative 10% แก้ปัญหาอะไร ใครคือกลุ่มผู้ใช้ มีประโยชน์ยังไง
	- Project Completion 10%
	- AI For Thai Ecosystem 5% 
		- ไม่เรียกใช้ API อื่นๆ 0%
		- อย่างน้อย 1 API 2%
		- ตั้งแต่ 2 APIs 5%
	- Presentation 10% 
ขั้นตอนการปฎิบัติตนในระหว่างการแข่งขัน
- Consult
	- Booth 1 Backend Day 1 18:00-23:00, Day 2 9:00-17:00
- Checkpoint
	- Checkpoint # 1
		- Draft Concept ของงานว่าจะมีกี่ endpoint ให้เขียนใน README.md หรือ Document ใดๆ
		- commit เอกสารเข้า Git Repo ให้เรียบร้อย
		- ตรวจวันเสาร์ 23:59น.
	- Checkpoint # 2
		- ยังปรับเอกสารได้ แต่ต้องปรับให้ตรง
		- ตรวจวันอาทิตย์ 8:00 น.
	- Checkpoint # 3
		- review backend แบบ final
		- ถ้ามีการปรับ API Spec โดนหักคะแนน
		- ตรวจวันอาทิตย์ 18:00น.
13.30 น.-14.30 น. กล่าวต้อนรับเปิดการแข่งขันการแข่งขัน AI Thailand Hackathon 2025
